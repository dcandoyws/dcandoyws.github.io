Token 机制的简析

Posted on 2018-09-17 | In 前端

token是什么

  token用计算机术语来说就是一个令牌，看了这个令牌术语的解析不是很懂，不如我们换个通俗的说法，暗号，我们都知道古代打仗时期军事信息的交流都是通过暗号验证的，所以token就是一个暗号，在数据传输之前，都要先进行校对暗号，不同的数据传输是对应不同的暗号的。

token的作用

  上面也说了token就是在数据传输前的进行验证的暗号，那我们来说一下具体可以应用于哪些场景：

  • 用户登录的身份验证
  • 防止表单重复提交
  • 防止CSRF攻击

这些应用场景我们下面会详细介绍下面我们来说一说token的优势，就是我们为什么要用它

token的优势

支持跨区域访问：

Cookie是不允许跨区域访问的，token机制是允许的，不过传输用户认证信息是通过HTTP头传输的

无状态（服务器可扩展性）：

token机制不需要在服务端存储session信息，因为token自身就包含了用户登录的所有信息，只需要在客户端Cookie或者本地存储状态信息

去耦：

不需要绑定到一个特定的身份验证方案，token可以在任何地方生成，只要在你的api被调用的时候，你可以进行token生成调用即可。

性能：

一次网络往返时间（通过数据库访问session信息）总比做一次HMACSHA256计算花的时间多得多

基于JWT的Token认证机制的实现

JSON Web Token是一个非常轻巧的规范，它允许我们使用JWT在用户和服务器之间保持可靠安全的信息传递

  • JWT的组成
    其实JWT就是一个字符串，这个字符串是由三个部分组成：
    • 头部（Header）
      这是用于描述该JWT最基本的信息，例如其类型及签名所用到的算法等。可以被表示成一个JSON对象

上面就是说类型是JWT，用到的算法是HS256。
头部也是需要Base64编码，编码后的字符串如下：
eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9

  • 载荷（payload）

    • iss:该JWT的签发者，是否使用是可选的
    • sub：该JWT所面向的用户，是否使用是可选的
    • exp：啥时候过期，这是一个时间戳，是否使用时可选的
    • aud：接收该JWT的一方，是否使用时可选的
    • iat：在什么时候签发的（unix时间），是否使用是可选的
      将以上json对象使用Base64编码可以得到以下的字符串（我们称之为载荷（payload））：

  • 签名
    将上面两个编码后的字符串用.连接在一起如下：
    eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJmcm9tX3VzZXIiOiJCIiwidGFyZ2V0X3VzZXIiOiJBIn0
    然后我们对上面字符串进行加密，加密用HS256算法。加密的时候我们需要提供一个密钥（secret）。如果我们使用mystar作为密钥的话，那我们就可以得到加密后的以下内容：
    rSWamyAYwuHCo7IFAgd1oRpSP7nzL7BF5t7ItqpKViM
    那好我们已经获得了JWT的三个组成部分，现在将它们用.连接起来：
    eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJmcm9tX3VzZXIiOiJCIiwidGFyZ2V0X3VzZXIiOiJBIn0.rSWamyAYwuHCo7IFAgd1oRpSP7nzL7BF5t7ItqpKViM
    这就是我们所需要的JWT字符串，这也会在我们的请求中出现，比如这样：
    https://your.awesome-app.com/make-friend/?jwt=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJmcm9tX3VzZXIiOiJCIiwidGFyZ2V0X3VzZXIiOiJBIn0.rSWamyAYwuHCo7IFAgd1oRpSP7nzL7BF5t7ItqpKViM
